# Vibe Annotator — V1 Design

> Click elements in your browser, describe what's wrong, and export structured feedback your AI agent can act on immediately.

---

## Stack Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| UI framework | Preact | ~3KB, full component model, best size/DX balance |
| Styling | Plain CSS, `va-` prefix | Zero deps, easy to replace with design system later |
| Build | Pre-bundled IIFE via tsup | Zero config for consumer — just install and add plugin |
| Server comms | REST middleware | Simple endpoints at `/__annotations/api/` |
| Screenshots | Viewport-width context | Full width, ~200px vertical padding around element |
| Reference images | Inline in textarea | Paste/drop into comment input, chat-style UX |
| Aesthetic | Terminal/bitmap pixel, minimal | Monospace, sharp borders, designed to be reskinned |

## Vite Plugin

### User-Facing API

```js
// vite.config.js
import { annotateUI } from 'vibe-annotator'
export default { plugins: [annotateUI()] }

// With options:
annotateUI({
  storagePath: '.ui-annotations',   // default
  position: 'bottom-right',         // icon position
  screenshotPadding: 200,           // vertical px context
})
```

### Plugin Hooks

- **`transformIndexHtml`** — Appends the pre-bundled IIFE `<script>` before `</body>`. Only in `serve` mode (not `build`).
- **`configureServer`** — Registers REST middleware under `/__annotations/api/`.

### Middleware Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| `GET` | `/api/annotations` | Load annotations.json |
| `POST` | `/api/annotations` | Save/update single annotation |
| `DELETE` | `/api/annotations/:id` | Remove annotation |
| `POST` | `/api/screenshot` | Save base64 PNG, return filename |
| `POST` | `/api/reference` | Save uploaded reference image, return filename |
| `GET` | `/api/images/:filename` | Serve saved images back to client |

### Storage

All data written to `.ui-annotations/` in project root:

```
.ui-annotations/
├── annotations.json
├── screenshot_001.png
├── screenshot_002.png
├── reference_001.png
└── reference_002.png
```

## Client Architecture

### State Model

Single state object via `useReducer` in root `App`, passed down via Preact context:

```ts
interface AppState {
  mode: 'inactive' | 'selecting' | 'annotating'
  annotations: Annotation[]
  activeAnnotation: string | null
  visionMode: boolean
  hoveredElement: HTMLElement | null
}
```

### Mode Transitions

```
inactive ──[click icon]──→ selecting
selecting ──[click element]──→ annotating (card opens)
annotating ──[save/cancel/delete]──→ selecting
selecting ──[click icon]──→ inactive
```

### Behaviour Per Mode

- **Inactive:** Zero event listeners on document. Only the floating icon exists in DOM.
- **Selecting:** `mousemove` + `click` listeners with `{ capture: true }` and `stopPropagation` to intercept host app events. Mousemove highlights elements. Click captures element data and transitions to annotating.
- **Annotating:** Card is open. Element selection paused. User fills feedback, saves/cancels/deletes.

### Element Exclusion

Container div gets `data-vibe-annotator` attribute. Selection logic filters out anything inside `[data-vibe-annotator]`.

## Element Selection

### Highlight Overlay

Single absolutely-positioned `<div>` repositioned on every `mousemove` via `getBoundingClientRect()`. Styled with dashed border + marching-ants CSS animation. Small tooltip near cursor shows `tag.className`.

### Selector Generation (priority order)

1. `[data-testid="value"]` — most stable
2. `#id` — unique by definition
3. Meaningful class path — walks DOM building short selector (e.g. `main > .hero-section > h1`), skipping generic classes
4. Structural path — `nth-child` fallback

Also generates XPath as fallback.

### Data Captured on Click

```ts
{
  selector: string
  xpath: string
  tag: string
  textContent: string      // first 100 chars
  boundingBox: DOMRect
  computedStyles: {        // 25 properties from spec
    fontSize, fontWeight, fontFamily, color, backgroundColor,
    lineHeight, letterSpacing, padding, margin, borderRadius,
    border, width, height, display, flexDirection, alignItems,
    justifyContent, gap, opacity, boxShadow, textAlign,
    textTransform
  }
}
```

## Screenshot Capture

When vision mode is on, `html2canvas` captures the area around the selected element at the moment of click (before the annotation card opens).

### Capture Area

- **Width:** Full viewport width
- **Height:** Element vertically centered with ~200px padding above and below
- **Scale:** 2x for retina
- **Element highlight:** Subtle outline drawn on the element in the screenshot so the AI knows which element was annotated
- **Target size:** Under 300KB per capture
- **Format:** PNG saved to `.ui-annotations/screenshot_NNN.png`

The annotation overlay is temporarily hidden during capture.

## Annotation Card

### Layout

```
┌──────────────────────────────────┐
│ #1  h1.page-title             ✕  │  header
├──────────────────────────────────┤
│ COLOR  SPACING  FONT  ALIGN     │  quick actions
│ REFERENCE  COMMENT               │
├──────────────────────────────────┤
│ ┌──────────────────────────────┐ │
│ │ Describe what should change  │ │  textarea + image
│ │                              │ │
│ │  [thumbnail.png ✕]           │ │  (paste/drop inline)
│ └──────────────────────────────┘ │
│ Priority: ■ HI  □ MED  □ LO    │
├──────────────────────────────────┤
│ ▶ Target styles                  │  collapsed by default
├──────────────────────────────────┤
│  [DELETE]       [CANCEL]  [SAVE] │  actions
└──────────────────────────────────┘
```

- Delete button only visible when editing an existing annotation (reopened via pin click)
- New annotations show Cancel + Save only

### Positioning

1. Prefer right of element, 12px gap
2. Fall back: left → below → above
3. Clamp to viewport bounds
4. Max-width: 360px, scrollable if overflow

### Quick Actions

Toggle buttons — multiple can be active simultaneously. Each adds intent string + relevant computed styles to the export. They augment the free-text comment, not replace it.

| Template | Label | Auto-captured styles |
|----------|-------|---------------------|
| Wrong color | `COLOR` | `color`, `backgroundColor` |
| Too much spacing | `SPACING` | `padding`, `margin`, `gap` |
| Wrong font | `FONT` | `fontFamily`, `fontSize`, `fontWeight`, `lineHeight` |
| Fix alignment | `ALIGN` | `display`, `flexDirection`, `alignItems`, `justifyContent`, `textAlign` |
| Doesn't match design | `REFERENCE` | Prompts image attach |
| General feedback | `COMMENT` | Free text only |

### Styles Diff Panel

Collapsed by default. Expands to show computed properties in two columns: `current | target`. User types into target column. Only modified rows included in export.

### Reference Image

Paste from clipboard (`Ctrl+V` / `Cmd+V`) or drag-and-drop into the textarea area. Shows as inline thumbnail with `✕` to remove. Sent as base64 to middleware, saved to `.ui-annotations/reference_NNN.png`.

## Pin Markers

After saving an annotation, a numbered pin appears at the element's top-left corner.

- Displays annotation number
- Click → reopens annotation card for editing
- Hover → tooltip with comment preview
- Tracks element position via `MutationObserver` + `ResizeObserver`
- Visual differentiation by priority (styling TBD with design system)

## Export Format

"Copy to Clipboard" generates structured markdown, ordered by priority (high first):

```markdown
I have 2 UI annotations to implement. Annotations are ordered by priority.

Screenshots and reference images are in .ui-annotations/

---

## Annotation 1 (HIGH) — h1.page-title

**Element:** `h1.page-title`
**Selector:** `main > .hero-section > h1`
**Current text:** "Welcome to Dashboard"

**Style changes:**
- font-size: 16px → 24px
- font-weight: 400 → 700

**Intent:** User flagged incorrect typography
**Comment:** "Match the Figma heading style"

**Screenshot:** .ui-annotations/screenshot_001.png
**Reference image:** .ui-annotations/reference_001.png

---
```

- No style diffs → shows "Current styles" with relevant computed values
- Vision mode off → omits screenshot/reference lines
- Button shows "Copied!" for 1.5s after click

## Styling Approach

Terminal/bitmap pixel aesthetic as the default skin:

- Monospace font (system monospace stack, Berkeley Mono if available)
- Sharp corners, 1px solid borders
- Dark background (`#0a0a0a`), light text (`#e0e0e0`)
- No gradients, no shadows, no border-radius
- All classes prefixed `va-` to avoid host app conflicts
- All styles in a single `styles.css` injected as `<style>` in the IIFE
- Component structure kept clean so styles can be replaced with a design system later

## Build Configuration

**tsup** produces two outputs:

1. **Plugin** → `dist/index.js` (ESM) + `dist/index.cjs` (CJS) — imported in `vite.config.js`
2. **Client** → `dist/client.iife.js` — inlined as string by the plugin

The plugin reads the client IIFE at build time and embeds it as a template literal. At runtime, `transformIndexHtml` injects it.

## V2 Considerations

The V1 architecture should not block these future features but does not implement them:

- **MCP server** — middleware can be extended to expose MCP protocol
- **Next.js/Webpack plugins** — client IIFE stays identical, only plugin wrapper changes
- **Design token awareness** — styles capture is already structured for comparison
- **Before/after comparison** — screenshot infrastructure is reusable
- **Annotation history** — storage layer can be extended with sessions
